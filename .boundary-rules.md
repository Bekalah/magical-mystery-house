# Hard Rule Boundary - Cathedral Project

## ⚠️ STRICT RULES - NO EXCEPTIONS

### 1. FREE AND OPEN SOURCE ONLY
- **MUST**: CC0-1.0 (Public Domain) license
- **MUST NOT**: Any commercial dependencies
- **MUST NOT**: Azure, AWS, Google Cloud, Microsoft
- **MUST NOT**: Any paid APIs or services
- **MUST NOT**: Any proprietary code

### 2. PNPM ONLY
- **MUST**: Use pnpm for all package management
- **MUST**: Include pnpm-lock.yaml file (required for reproducible builds)
- **MUST NOT**: npm, yarn, or any other package manager
- **MUST NOT**: npm references in scripts, workflows, or configs
- **MUST NOT**: package-lock.json, yarn.lock, or other lockfiles

### 3. QUALITY STANDARDS
- **MUST**: Meet or exceed design/tools/work quality
- **MUST NOT**: Inferior dependencies or tools
- **MUST NOT**: Degraded or deprecated packages
- **MUST NOT**: Unmaintained or abandoned packages

### 4. NO COMMERCIAL CREEP
- **MUST**: Reject any commercial dependencies immediately
- **MUST**: Remove any commercial code found
- **MUST**: Replace with free/open source alternatives
- **MUST NOT**: Allow any exceptions

### 5. COMPLETE CORRECTNESS
- **MUST**: Fix everything, no matter how minor
- **MUST**: Update all incorrect configurations
- **MUST**: Remove all deprecated patterns
- **MUST NOT**: Leave anything "just out there"
- **MUST NOT**: Assume something "won't hurt"

## Enforcement

These rules are enforced by:
- Automated scripts that check all files
- Pre-commit hooks
- CI/CD pipeline checks
- Regular audits

**Violations will be automatically fixed or rejected.**

---

**This is a healing space. It must remain free, open, and correct.**
